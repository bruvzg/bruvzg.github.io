Complex text layouts progress report #2:

Changes to the Godot Font and Theme:

Since font handling was moved to TextServer, some substantial changes were made to the Godot Font and related classes:

BitmapFont, DynamicFont and DynacmicFontData were removed and replaced with universal Font and FontData (which are backed by TextServer), this provides cleaner font fallback/substitution and possibility for custom text servers to expose different types of font without interface changers, for example direct access to the system fonts.

New Font class provides new functions to draw and measure multiline text, and apply alignment.

Font and outline size is now argument of the draw functions and a theme constant, rather than property of the Font instance. This allow changing font size for individual draw calls, controls, or spans of text in the RichTextLabel control without creating new Font instances.

Functions for loading font data from the memory buffer are exposed to GDScript and GDNative.

New, smarter font substitution system is added:

  • Each font data have associated list of supported scripts (writing systems) and languages. For a TrueType/OpenType fonts, script list is populated automatically from the OS2 table, but it's not always precise.
  • Script and language support can be overridden by user.
  • For each run of text with specific script and language, TextServer will try to use fonts in the following order:
    • Fonts with both script and language supported.
    • Fonts with script supported.
    • Rest of the fonts.

Here're few cases of manual override:

  1.  Many latin fonts have limited set of Greek characters for use in scientific texts (and such fonts usually have Greek script support flag set in the OS2 table), but it's not always enough to display Greek text. Adding separate font with full Greek support, and disabling Greek support in the main font will prevent TextServer from mixing characters form different fonts.
  2. TrueType/OpenType font tables do not have flags for rare/ancient scripts (e.g Egyptian hieroglyphs), enabling script support manually will speed up font substitution.
  3. Some languages use the same script, but different font styles (e.g Kufic, Naskh and Nastaʼlīq writing styles preferred for writing different Arabic languages/ in different regions), setting language overrides allow to seamlessly use same font stack for the text in different languages and get desired style.

Additional new Font and TextServer's BiDi and shaping features can work with variable fonts (see https://github.com/godotengine/godot/pull/43030 for variable font support PR):

UI mirroring:

User interface for the right-to-left written languages should flow from right-to-left to ensure the content is easy to understand. UI "mirroring" provides convenient way to archive this without designing separate interfaces for the RTL and LTR languages.

Each Control have "layout_direction" property to control mirroring, it can be set to "inherited" (use same direction as the parent control, same as auto for the root control), "auto" (direction is determined based on current locale) or forced to RTL or LTR.

On the screenshots above, green and blue "compass" controls are forced to LTR and RTL layout respectively, red container is set to "auto" and the rest of UI use "inherited" (default setting) direction.

RTL layout direction mirrors margins/anchors, order of the child controls in the containers horizontal alignment of the text and internal elements of controls (e.g. OptionBox dropdown button, List column order, Tree item icons and connecting line alignment, e.t.c.), coordinate system is not mirrored and non-UI nodes (2D Sprites e.t.c) are not affected.

Structured text BiDi override:

Unicode BiDi algorithm is designed to work with natural text, but it's incapable of handling text with implicit, fixed higher level order line file names, URIs, email addressed, regular expressions or source code.

structured_text_bidi_override property and _structured_text_parser callback is added to the all controls with a text).

For example path for this directory structure will be displayed incorrectly (top LineEdit), structured text override add a callback function to split text into segments, then BiDi is applied to each of them individually (bottom LineEdit).

Custom callback provides a way to override BiDi for other structured text types. For example, following code will split text by ":", apply BiDi to each part and display the in reverse order, override can be used with any control, including input fields (LineEdit, TextEdit):

func _structured_text_parser(args, text):
    var ranges = []
    var offset = 0
    for t in text.split(":"):
         ranges.push_front(Vector2i(offset, offset + t.length())) # Add text
         ranges.push_front(Vector2i(offset + t.length(), offset + t.length() + 1)) # Add ":"
         offset = offset + t.length() + 1
    return ranges

Other changes to Godot controls:

All controls with the text have following properties added to control BiDi and font related features:

language property (only for controls with text):
Overrides locale for the control, and also controls language specific line breaking rules, shaping, and font substitution.

opentype_features property (only for controls with text):
Controls OpenType font features for the control: e.g. ligature types to use, number styles, small caps etc. (Full list of tags: https://docs.microsoft.com/en-us/typography/opentype/spec/featuretags)

Control and Window functions add_theme_font_size_override, get_theme_font_size, has_theme_font_size, has_theme_font_size_override and Theme functions: clear_font_size, get_font_size, get_font_size_list, has_font_size, set_font_size
Control font size in the similar manner to existing function for the Font, value -1 can used as unset/default (have the same effect as null for the font).
