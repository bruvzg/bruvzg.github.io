Complex text layouts progress report #2:

Introduction:

1. Unicode BiDi algorithm is designed to work with natural text, but it's incapable of handling text with implicit, fixed higher level order line file names, URIs, email addressed, regular expressions or source code.

2. Not only text but the whole UI in RTL languages should flow from right-to-left. UI "mirroring" provides convenient way to archive this without designing separate interfaces for RTL and LTR languages.

Changes to the Godot Font and Theme:

Since font handling was moved to TextServer, some substantial changes were made to the Godot Font and related classes:

BitmapFont, DynamicFont and DynacmicFontData were removed and replaced with universal Font and FontData (which is backed by TextServer), this provides cleaner font fallback/substitution and possibility for custom text servers to expose different types of font without interface changers, for example access to the system fonts.

Font now provides new functions to draw and measure multiline text, and apply alignment.

Font size is not argument of the draw function and a theme constant, rather than property of the Font instance. This allow changing font size for individual draw calls, controls, or spans in RichTextLabel  without creating new Font instances.

Functions for loading font data from the memory buffer is now exposed to GDScript and GDNative.
